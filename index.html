סיכום חלק א׳ – מה קרה כאן?

כדי להבין מה קרה, צריך להבין איך הזיכרון עובד:

משתנים פרימיטיביים – וה- Stack:

כשמגדירים משתנה פרימיטיבי כמו int, char או bool – המחשב יודע בדיוק כמה זיכרון לשמור לו. לכן הערך עצמו נשמר ישירות בזיכרון שנקרא Stack:

STACK
int num = 10     [4 bytes]
char c = 'A'    [2 bytes]
bool flag = true   [1 byte]

כל משתנה מקבל את המקום המדויק שהוא צריך, והערך שלו מאוחסן שם ישירות.

מערכים – וה- Heap:

אבל מה קורה עם מערך? אנחנו יודעים שתא מטיפוס int תופס 4 בתים – אבל לא ידוע מראש כמה תאים יהיו. לכן אי אפשר לשמור מערך ב- Stack.

הפיתרון: המערך עצמו נשמר בזיכרון גמיש יותר שנקרא Heap. וב- Stack נשמרת כתובת – שמייצגת היכן נמצא המערך ב- Heap:

STACK
int num = 10     [4 bytes]
char c = 'A'    [2 bytes]
bool flag = true   [1 byte]
myArr1 = 0x4A2F   [4 bytes]
⟶
HEAP
... other memory ...
address 0x4A2F:
[10][20][30][40][50]
... other memory ...

המשתנה myArr1 ב- Stack מחזיק למשל, את הכתובת 0x4A2F – כלומר "המערך שלי נמצא בכתובת הזו ב- Heap".

מה קורה כשכותבים myArr2 = myArr1?

השורה הזו מעתיקה את הכתובת – לא את המערך. עכשיו גם myArr2 מצביע לאותו מקום ב- Heap:

STACK
myArr1 = 0x4A2F   [4 bytes]
myArr2 = 0x4A2F   [4 bytes]
⟶ ⟶
HEAP
address 0x4A2F:
[10][20][30][40][50]

שני המשתנים מצביעים לאותו מערך. שינוי דרך אחד משפיע על השני – כי הם מצביעים על אותו מקום בזיכרון.
